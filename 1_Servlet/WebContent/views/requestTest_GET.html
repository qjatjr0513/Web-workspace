GET 방식으로 요청 후 응답페이지 받아보기

특징1. GET방식으로 요청하는건 URL Header영역에 데이터들을 포함시켜 요청함
=> 사용자가 입력한 값(데이터)들이 url에 노출됨
=> 보안유지가 불가능함
=> 즉, 로그인이나 회원가입같은 경우 get방식은 부적합하다

특징2. Header영역은 전송하는 길이에 제한이 있음
=> 방대한 데이터를 담았을 경우 초과된 데이터는 절단되어서 넘어감
=> 즉, 게시판 작성 같은 경우 get방식이 부적합 함

특징3. 장점이라고 한다면 url에 데이터가 노출되기 때문에 즐겨찾기 기능 가능
=> 즉 검색기능 같은 경우 get방식에 적함함.
