POST 방식으로 요청 후 응답 페이지 받아보기

특징 1. POST방식으로 요청하는건 URL의 Body영역에 데이터를 포함시켜 요청
=> 사용자가 입력한 값(데이터)들이 URL에 노출되지 않음
=> 보안 유지가 가능
=> 즉, 로그인이나 회원가입 같은 경우 POST방식이 적합함

특징 2. Body영역은 전송하는 길이에 제한이 없음
=> 즉, 게시판 작성같은 경우 post방식이 적합함

특징 3. 즐겨찾기는 가능하나 전달되는 데이터가 URL에 노출되어있지 않음(기존에 봤던 응답화면을 볼수 없음)

특징 4. 최대요청받는 시간(time out)이 존재해 페이지 요청, 기다리는 식간이 있다.
